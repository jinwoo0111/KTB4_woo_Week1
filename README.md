### 풀스택 1주차 과제

속성, 메소드, 상속을 알아보기 위한 과제 주제로 '운동 선수 아바타 만들기 프로그램을 제작하였다.

#### 클래스 상속 구조
1. Athelete
2. MartialArts (투기 종목) , Ball(구기 종목)
3. MartialArts 상속 -> Boxing(복싱), Jiujitsu(주짓수) / Ball 상속 -> Baseball(야구), Basketball(농구)

#### 객체별 속성
Athlete
 name : String (아바타 이름)
 age : int (아바타 나이)

MartialArts (투기)
 size : String (체급 - 플라이급/미들급/헤비급)

Boxing (복싱)
 stance : String (스탠스 : 사우스포/오소독스)

Jiujitsu (주짓수)
 style : String (스타일 : 브라질리언/유러피안)

Ball (구기)
 handed : String (주손 - 왼손잡이 / 오른손잡이)

Baseball(야구)
 pos : String (포지션 - 투수/타자)

Basketball(농구)
 pos : String (포지션 - 가드/포워드/센터)

#### 회고
05.17
최근에 코딩 테스트를 준비하며 절차지향으로만 코드를 작성해와서, 개발을 위한 코딩을 오랜만에 해보았다. 
객체지향적 관점에서 코드를 작성하는 것이 어색해서 주제를 선정하고 구현하는데 시간이 좀 걸렸다. 다 작성하고 보니, 
변수들에 대한 상속은 있는데 동작에 대한 상속을 생각하지 못했다. 이 부분에 대해서 적응과 공부가 더 필요해 보인다.

05.25
자바 문법과 코드 작성에 부족함을 느껴서, 주말동안 교재들을 다시 복습하고 교재 내용들에 관련해서 AI한테 구현 문제를 받으면서 코드를 짜보았다. 그 과정에서 클래스와 상속, 인터페이스, 접근 제어자 등에 대해서 더 잘 알 수 있었고 1주차 과제에 부족한 부분들이 보였다.
이후 코드 리뷰에 대해서 고민해보고 수정 사항들을 반영했다. 

각 클래스(Athlete -> MartialArt, Ball -> (Boxing, Jiujitsu) , (Baseball, Basketball) )의 변수들에 접근 제어자를 아예 적용하지 않았어서, private으로 설정하고 getter 메소드들 두었다.
또한 입력은 Main 클래스에서만 받도록 설정하여, 객체 생성할 때 인자로 넘겨주었다.
클래스 정보의 출력의 경우, 2차 상속까지 이어져있어서, 부모 클래스의 출력 메서드를 오버라이딩하여 추가 정보들을 출력할 수 있도록 수정하였고, 임의의 입력 숫자를 통해 객체를 특정하여 생성했는데, 이를 enum을 통해 따로 관리하도록 하여 명확하게 무엇인지 드러나도록 변경했다.
